# Rules for AI Agents

This file contains the absolute guidelines and rules that artificial intelligence agents (like me) must follow when interacting with this repository.

## Absolute Rules

1. **NEVER MAKE COMMITS.**
   - From this version of the project onwards, the responsibility to execute commands like `git commit` or `git push` belongs **strictly and solely to the human user**.
   - The agent can stage files (`git add`), edit files, run commands, and suggest commit messages, but the final execution of the commit is done by the user.

2. **Follow the Specification (SPEC).**
   - All architectural and technical decisions must strictly follow the documents present in the `/docs` folder.
   - Modifications to the main plan must be previously discussed and recorded in the documentation.

3. **Documentation Architecture.**
   - The root document `SPEC.md` serves only as a summary/index.
   - Technical details (Hardware, Architecture, Database, Infrastructure, etc.) must be segmented into specific files within `/docs`.
